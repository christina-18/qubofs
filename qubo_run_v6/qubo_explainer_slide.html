QUBO 遺伝子選択の仕組み
「ちょうど良い K=20 個」を組合せ最適化で選ぶ

単純な順位ソート (DE_top) でも LASSO でもなく、「効く × 似てない × ちょうど20個」を同時最適化するのが QUBO

H(x) = −s⊤x + γ · x⊤R x + λ (1⊤x − K)2
① relevance を最大化
各遺伝子の「効きそう度」 si (= |t-stat|² など) を 選んだ遺伝子で総和。「MS と HD で発現が違う遺伝子」が選ばれやすくなる
② redundancy を最小化
遺伝子ペアの相関 Rij = |corr| を 選んだペアでペナルティ加算。互いに似た遺伝子を避ける (= 多様性確保)
③ ちょうど K 個
(Σxi − K)² がゼロになるよう罰則。K=20 を超えても足りなくても「コスト」がかかる

x ∈ {0,1}n: 各遺伝子を選ぶ(1)/選ばない(0) の二値変数。s, R は train donor のみから推定。 H(x) を最小化する組合せ x* が「最良の K 遺伝子集合」となる。

解き方: Simulated Annealing
x = ランダムな初期解 (K個 1)
T = T_start (高温; 探索広い)
while T > T_end:
   for i in 全遺伝子:
      ΔH = H 変化量 (i を flip した時)
      if exp(−ΔH/T) > rand():
         flip x[i] # 受理
   T *= 0.99 # 冷却
# T 高い時: 山も下る (探索)
# T 低い時: 良い方向のみ (収束)

高温では悪い方向にも一定確率で動き、局所最適から脱出。冷却で徐々に最良解へ収束。

QUBO の利点
  • 冗長性を明示制御: 単純ソート (DE_top) では同じ pathway の遺伝子ばかり選ばれがち。QUBO は多様な遺伝子を確保
  • cardinality 制約が直接書ける: 「ちょうど 20 個」が損失関数に組み込み可能 (LASSO は λ チューニングで間接的にしか制御できない)
  • relevance と redundancy のバランスを γ, λ で調整可能: cell type ごと最適化できる
  • 量子アニーラ (D-Wave) 互換: 二値最適化で書けば量子計算機にそのまま実装可能。現在は古典 SA でも十分高速 (~3 秒/instance)
  • 本研究で実証: AUC 0.781 ± 0.041 で他 4 手法を上回った (主指標 AUC、補助指標 MCC, F1 で勝)
